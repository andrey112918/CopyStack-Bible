# Research Automation Architecture

**Status:** Active Development  
**Owner:** CopyStack  
**Last Updated:** 2026-07-09

---

# Overview

Research Automation is the deterministic processing pipeline that transforms raw media into reusable structured knowledge.

Unlike future AI layers, Research Automation does not attempt to understand why a video performs well.

Its responsibility is to prepare reliable, reusable, explainable data that higher-level intelligence systems consume.

Research Automation is therefore the infrastructure layer of the Research Center.

---

# Design Goals

The architecture follows several design principles.

- Deterministic outputs
- Reusable data
- Explainable processing
- Stable identifiers
- Versioned artifacts
- Independent modules
- Extensible pipeline
- AI-ready foundations

Every processing stage should produce artifacts that can be consumed independently.

No stage should depend on hidden internal state.

# Processing Pipeline

Each research session follows the same deterministic pipeline.

```
Reference Material

↓

Research Job

↓

Media Import

↓

Storage

↓

Media Profile

↓

Transcript

↓

Frames

↓

Keyframes
```

Future semantic systems never analyze raw media directly.

They consume the structured outputs generated by this pipeline.

---

# Layer 1 — Research Job

Purpose:

Coordinate the complete research lifecycle.

Responsibilities:

- Job creation
- Stable IDs
- Source registration
- Progress tracking
- Status management
- Error handling
- Duplicate detection

Output:

Research Job artifacts.

---

# Layer 2 — Media Import

Purpose:

Normalize every supported media source.

Supported:

- Local video
- Metadata-only YouTube
- Future cloud providers

Responsibilities:

- Source validation
- Duplicate handling
- Import manifests
- Source metadata

Output:

Registered media source.

---

# Layer 3 — Storage

Purpose:

Persist every artifact deterministically.

Current artifacts include:

- job.json
- sources.json
- metadata.json
- import_result.json
- manifests
- indexes

Storage never performs analysis.

It only preserves results.

---

# Layer 4 — Media Analysis

Purpose:

Describe imported media.

Responsibilities:

- Media Profile
- Capability Detection
- Processing Manifest
- Ready State
- Validation

Current limitation:

Stream probing is intentionally lightweight and will be expanded during Media Probe Upgrade.

---

# Layer 5 — Transcript Foundation

Purpose:

Create the textual representation of media.

Current capabilities:

- Transcript model
- Segment model
- Word model
- Transcript quality
- Manual registration
- Transcript storage

Future:

Automatic speech recognition.

---

# Layer 6 — Frame Foundation

Purpose:

Represent visual information.

- Timeline mapping
- Thumbnail generation

# Layer 7 — Scene Foundation

Scene timeline.

# Deterministic Processing

Research Automation intentionally avoids AI reasoning.

Given identical inputs:

- IDs remain stable.
- Storage remains identical.
- Manifests remain identical.
- Processing order remains identical.

This deterministic behavior simplifies testing, debugging and reproducibility.

---

# Knowledge Boundary

Research Automation does **not** generate knowledge.

It prepares structured evidence.

Knowledge generation begins only after the semantic layers are implemented.

This separation ensures that:

- infrastructure remains reusable
- semantic reasoning can evolve independently
- AI modules remain replaceable

# Architectural Principle

Research Automation exists to answer:

> **"What is inside this video?"**

Future Intelligence Layers answer:

> **"Why does this video work?"**

Keeping these responsibilities separate is one of the core architectural decisions of the CopyStack AI Operating System.
